# 공개 상태판

One-Minute Three Kingdom은 현재 공개 출시작이 아니라 **제한 초대 베타 /
프로토타입**입니다.

핵심 경험은 이미 테스트 중입니다: SSH로 접속해 TUI 보고서를 읽고, 짧은 명령을
내리고, 접속 밖에서 진행된 세계를 다음 방문에 다시 확인합니다. 하지만 이 공개
레포는 서비스 약관, 전체 source release, uptime 보장, 완성품 광고가 아닙니다.
README와 Markdown, 공개 이미지로 현재 상태를 솔직하게 보여주는 showcase입니다.

## 한눈에 보기

| 항목 | 현재 상태 |
| --- | --- |
| 접근 방식 | 초대받은 테스터에게만 비공개 접속 정보 제공 |
| 주 인터페이스 | SSH 위에서 실행되는 TUI |
| 가입 방식 | 셀프서비스 가입 아님 |
| 플레이 규모 | 작은 커뮤니티 테스트를 우선 |
| 데이터 보존 | 베타 중 초기화 가능 |
| 웹 클라이언트 | 현재 공개 약속 없음 |
| 과금 | 현재 공개 과금 동선 없음 |
| 공개 레포 역할 | 소개, 상태판, 참여 안내, 공개 가능한 이미지 보관 |

공개 문서의 접속 예시는 항상 placeholder입니다.

```sh
ssh <player>@<beta-host>
```

## 지금 검증하는 것

- 1분 단위의 짧은 접속이 충분한 진행감으로 이어지는가.
- 보고서, 소문, 청원, 장부가 세계 상태를 읽기 쉽게 만드는가.
- 비동기 경쟁이 과열보다 공동체의 기억으로 남는가.
- 새 플레이어가 자리를 비운 동안의 변화를 따라올 수 있는가.
- 운영자가 작은 테스트를 지속 가능한 부담으로 운영할 수 있는가.

## 현재 기대해도 되는 것

- 텍스트 중심의 느린 전략 롤플레잉 감각
- 짧게 접속하고 빠져나오는 플레이 루프
- 베타 테스트 중 바뀌는 명령어, 도움말, 밸런스
- 공개 가능한 수준의 스크린샷, 다이어그램, 상태 설명
- 작은 커뮤니티를 우선하는 초대 운영

## 아직 기대하면 안 되는 것

- 누구나 즉시 접속하는 공개 서비스
- 완성된 웹 UI 또는 모바일 앱
- 영구 캐릭터 보존 보장
- 공개적인 uptime 또는 지원 응답 시간 보장
- 유료 특전, 프리미엄 접속권, 상점 중심 성장
- 모든 내부 디자인 문서나 운영 로그 공개

## 알려진 한계

- 온보딩 문구는 아직 불완전하며 자주 바뀔 수 있습니다.
- 일부 명령어 이름과 도움말은 테스트 사이에 변경될 수 있습니다.
- 밸런스, 진행 속도, 보상은 최종 상태가 아닙니다.
- 베타 시즌 중 플레이어 상태, 저장 데이터, 공개 기록이 초기화될 수 있습니다.
- 공개 문서와 이미지가 실제 플레이보다 늦게 갱신될 수 있습니다.

## 공개 가능한 업데이트

이 레포에 올릴 수 있는 상태 업데이트는 다음 범위로 제한합니다.

- 베타 단계 변화
- 공개 신청 방식 변화
- 공개 가능한 플레이 감각과 학습 요약
- README나 이미지와 실제 게임 상태의 차이를 줄이는 수정
- 보안과 운영 정보를 노출하지 않는 수준의 로드맵 조정

실제 host, 운영 계정, invite-only 공간, moderation 세부 사항, 내부 작업 경로는
공개 상태판에 넣지 않습니다. 공개/비공개 경계는
[public-boundary.md](public-boundary.md)를 따릅니다.

- 공개 문서가 현재 베타 상태와 어긋나면 빠르게 수정합니다.
