# 베타 참여

**One-Minute Three Kingdom**은 현재 제한 초대 베타로 운영됩니다.

베타를 작게 여는 이유는 희소성 연출이 아닙니다. 터미널-first 비동기 MUD는 읽기 쉬운 공동 기록, 차분한 온보딩, 플레이어와 운영자 사이의 신뢰가 있어야 오래 갑니다. 너무 빨리 커지면 커뮤니티 게임의 장점인 "서로의 선택을 알아보는 감각"이 먼저 닳을 수 있습니다.

게임이 어떤 세계인지 먼저 알고 싶다면 [세계관](./world.md)을, 실제 명령 흐름을 보고 싶다면 [플레이 소개](./how-it-plays.md)를 읽어 주세요.

## 신청 전 확인할 것

현재 공개 레포는 랜딩페이지 역할을 하는 showcase입니다. 실제 source repo, 운영 도구, 미공개 시나리오, 접속 정보는 공개하지 않습니다.

베타 참여자는 아래 전제를 받아들일 수 있어야 합니다.

- 접속 표면은 베타 구간에 따라 SSH 터미널, TUI, 또는 운영자가 공개하는 Web 접속 정보로 안내될 수 있습니다.
- 모두가 동시에 접속해 있어야 하는 게임이 아닙니다.
- 시즌 중 초기화, 명령 이름 변경, 콘텐츠 폐기가 일어날 수 있습니다.
- 공개 문서의 명령과 화면 예시는 플레이 감각을 설명하기 위한 예시이며, 실제 빌드와 다를 수 있습니다.
- 비공개 host, 계정, 초대 토큰, 운영 메모는 공개 이슈나 스크린샷에 올리지 않습니다.

접속 명령이 필요할 경우, 운영자가 초대 안내에서 현재 유효한 정보를 따로 전달합니다. 공개 문서에서는 확정 주소처럼 쓰지 않습니다.

```sh
ssh <player>@<beta-host>
```

## 신청이 열리면

공개 신청이 열리면 이 문서나 README의 신청 경로가 실제 링크로 교체됩니다.

그때 요청에 포함하면 좋은 내용:

- 사용하고 싶은 플레이어 이름
- 시간대 또는 주로 접속할 수 있는 시간
- SSH/TUI 사용 경험 여부
- 관심 있는 플레이 방식: 정치, 보급, 경쟁, 글쓰기, 탐험, 거친 부분 테스트
- 터미널 플레이에 영향을 주는 접근성 요구
- 피드백을 남기기 편한 공개 또는 비공개 연락 방식

비밀번호, private key, 실제 네트워크 세부 정보, 민감한 개인정보는 보내지 마세요.

## 초대가 오면

현재 테스트 구간에 자리가 있으면 다음 흐름으로 진행됩니다.

1. 운영자가 현재 베타 slice와 신청 내용이 맞는지 확인합니다.
2. 승인된 연락 경로로 비공개 초대 정보를 받습니다.
3. 운영자가 공개하는 접속 정보와 배정된 플레이어 이름으로 접속합니다.
4. 첫 접속에서는 온보딩, 보고, 명령, 장부 흐름을 확인합니다.
5. 혼란, 버그, 사회적 마찰은 초대 안내에 적힌 공개 또는 비공개 채널로 제보합니다.

접속은 보장되지 않습니다. 조용한 "아직은 아닙니다"는 보통 현재 테스트에 더 작은 인원이 필요하다는 뜻입니다.

## 좋은 피드백

좋은 베타 피드백은 "재미있다"나 "불편하다"에서 한 걸음 더 들어갑니다. 플레이어가 실제로 무엇을 봤고, 무엇을 눌렀고, 무엇을 잃거나 얻었다고 느꼈는지가 중요합니다.

특히 이런 피드백이 좋습니다.

- 첫 접속 1시간 안에 무엇을 해야 할지 알 수 있었는지
- [command]report[/command], [command]orders[/command], [command]choose[/command], [command]ledger[/command] 흐름이 자연스러웠는지
- 선택의 비용과 보상이 충분히 읽혔는지
- 색이 들어간 짧은 토큰이 도움이 되었는지, 또는 문장을 방해했는지
- 다른 플레이어의 흔적을 읽고 다음 행동을 정할 수 있었는지
- 경쟁, 장난, 정보 비대칭이 커뮤니티 건강을 해칠 위험은 없었는지
- 버그, 계정 문제, 괴롭힘, 악용 가능성이 있었는지

가능하면 "무엇을 시도했다 -> 무슨 일이 일어났다 -> 무엇이 헷갈리거나 좋았다" 순서로 남겨 주세요.

## 베타 etiquette

- 비공개 접속 정보는 비공개로 유지합니다.
- 초대 전용 hostname, 계정, moderation note, 미공개 콘텐츠가 보이는 스크린샷을 공개하지 않습니다.
- 게임 안의 경쟁은 가능한 한 fiction으로 다룹니다.
- 악용 가능성, 계정 문제, 괴롭힘은 비공개로 신고합니다.
- 공개 피드백에는 다른 플레이어의 민감한 정보나 운영 단서를 넣지 않습니다.
- 피드백은 실제 플레이 경험 중심으로 남깁니다.

## 왜 초대제인가요

이 게임은 reach보다 community를 먼저 만들려고 합니다. 느린 베타는 짧은 터미널 접속만으로도 기억에 남는 공적 역사, 반복해서 마주치는 인물, 부담 없는 협력과 경쟁이 생기는지 확인하기 위한 장치입니다.

베타의 질문은 단순합니다.

```text
1분 뒤에 나가도,
다음 접속 때 다시 펼치고 싶은 장부가 남는가.
```
